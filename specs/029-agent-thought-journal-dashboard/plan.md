# Implementation Plan: Agent Thought Journal Dashboard

**Input**: Feature specification from `/specs/029-agent-thought-journal-dashboard/spec.md`

## Summary

The Agent Thought Journal Dashboard implements a real-time observability layer that streams critical risk metrics and agent reasoning from the Python Orchestrator to the frontend. This will be achieved using an asynchronous WebSocket-based telemetry stream, architected to ensure zero impact on the trading execution latency budget.

## Technical Context

**Language/Version**: Python 3.11, JavaScript (ES6+)  
**Primary Dependencies**: FastAPI (WebSockets), WebSockets API (Frontend), asyncio, Pydantic  
**Design Pattern**: Asynchronous Producer-Consumer (via `asyncio.Queue` or fire-and-forget tasks)  
**Target Platform**: Linux (Docker)  
**Performance Goals**: < 100ms end-to-end telemetry latency; 0ms impact on execution hot path  
**Constraints**: Zero blocking of the GIL; decoupling of telemetry from trade execution  
**Scale/Scope**: Real-time streaming of risk and reasoning for all active signals; multi-client support

## Strategy

1.  **Backend Telemetry**: Enhance `TelemetryService` to include a non-blocking `broadcast` method. This method will use `asyncio.create_task` to ensure the call returns immediately without waiting for the network or IO, fulfilling the "fire-and-forget" requirement.
2.  **Dashboard WebSocket**: Update `DashboardService` to handle WebSocket connections. Use a simple broadcast mechanism (maintaining a set of active connections) to push updates to all connected clients.
3.  **Agent Integration**: Add telemetry hooks to the Orchestrator and individual agents (Bull, Bear, SEC) to push their "thoughts" as they are generated.
4.  **Frontend HUD**: Redesign the "Thought Journal" and "Risk HUD" on the dashboard to handle high-frequency updates and provide a clean, visual representation of the bot's internal state.
5.  **Performance Guard**: Benchmark the telemetry stream under load to ensure it does not introduce jitter into the gRPC execution path.
