# Implementation Plan: Multi-Agent Arbitrage Engine

**Branch**: `005-arbitrage-multiagent-engine` | **Date**: 2026-03-29 | **Spec**: [specs/005-arbitrage-multiagent-engine/spec.md]
**Input**: Drawing technical plan using Python 3.12+, LangGraph for orchestration and FastMCP for tools. WebSocket for Polygon.io, yfinance for history, Trading 212 wrapper, Risk Engine with VaR (Monte Carlo) and Kelly Fractional. SQLite persistence and QuantStats HTML reports.

## Summary

This feature implements a high-fidelity statistical arbitrage engine using a multi-agent approach. It leverages LangGraph to coordinate specialized agents (Bull, Bear, News Analyst) that validate signals generated by a real-time statistical monitor. The system ensures capital preservation through a sophisticated risk engine (VaR, Kelly) and maintains a "Total Auditability" regime via a Thought Journal and automated QuantStats reporting.

## Technical Context

**Language/Version**: Python 3.12+
**Primary Dependencies**: LangGraph, FastMCP, pandas, statsmodels, polygon-api-client, yfinance, python-telegram-bot, QuantStats, scipy, tenacity
**Storage**: SQLite (Z-score history, Virtual Pie state, Trade Ledger, Thought Journal)
**Testing**: pytest (unit and integration)
**Target Platform**: Linux (amd64)
**Project Type**: Multi-agent trading system / MCP Service
**Performance Goals**: <10s decision latency (from signal to execution); <1% slippage simulation in Shadow mode.
**Constraints**: Strict NYSE/NASDAQ operating hours (14:30-21:00 WET); <2% risk per trade.
**Scale/Scope**: 20 initial arbitrage pairs; Real-time WebSocket monitoring.
