# Data Model: Multi-Agent Arbitrage Engine

## Entities

### ArbitragePair
Represents a pair of cointegrated assets being monitored.
- `id` (UUID, PK)
- `ticker_a` (string): Primary asset.
- `ticker_b` (string): Hedge asset.
- `hedge_ratio` (float): Regression coefficient from OLS.
- `is_cointegrated` (boolean): Current status based on ADF test.
- `last_adf_pvalue` (float): p-value from the latest cointegration test.
- `status` (string): 'Active', 'Suspended', 'High-Volatility'.

### Signal
A statistical trigger generated by the monitor.
- `id` (UUID, PK)
- `pair_id` (UUID, FK): Link to `ArbitragePair`.
- `z_score` (float): Current deviation from the mean.
- `timestamp` (datetime): Time of detection.
- `confidence_score` (float): Output from the multi-agent debate (0.0 to 1.0).
- `news_sentiment` (float): Sentiment score from the Gemini analyst (-1.0 to 1.0).

### TradeRecord
Execution history for both Live and Shadow modes.
- `id` (UUID, PK)
- `signal_id` (UUID, FK): Link to the triggering `Signal`.
- `pair_id` (UUID, FK): Link to `ArbitragePair`.
- `direction` (string): 'Long-Short' or 'Short-Long'.
- `entry_timestamp` (datetime)
- `exit_timestamp` (datetime, nullable)
- `size_a` (float): Amount of ticker_a.
- `size_b` (float): Amount of ticker_b.
- `entry_price_a` / `entry_price_b` (float)
- `exit_price_a` / `exit_price_b` (float)
- `is_shadow` (boolean): True for simulation mode.
- `status` (string): 'Open', 'Closed', 'Canceled'.
- `pnl` (float): Realized or hypothetical profit/loss.

### ThoughtJournal
Persistent audit log of the AI reasoning process.
- `id` (UUID, PK)
- `signal_id` (UUID, FK): Link to `Signal`.
- `bull_argument` (text): Summary of the Bull Agent reasoning.
- `bear_argument` (text): Summary of the Bear Agent reasoning.
- `news_analysis` (text): Contextual info from the News Analyst.
- `final_verdict` (text): Consolidated decision explanation.
- `shap_values` (JSON): Feature importance scores for transparency (Auditability Principle).

### VirtualPortfolio (Virtual Pie)
The application-level state of asset allocations.
- `id` (UUID, PK)
- `ticker` (string)
- `allocated_amount` (float): Intended position size.
- `current_value` (float): Real-time value of the position.
- `last_reconciliation` (datetime): Last time state was checked against brokerage API.

## Relationships
- `ArbitragePair` 1:N `Signal`
- `Signal` 1:1 `ThoughtJournal`
- `Signal` 1:N `TradeRecord` (multiple attempts or legs)
- `TradeRecord` N:1 `ArbitragePair`
