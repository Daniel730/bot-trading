# Implementation Plan: Paper-Trading Readiness Blockers

**Branch**: `036-paper-readiness-blockers` | **Date**: 2026-04-19 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/036-paper-readiness-blockers/spec.md`

## Summary

Three surgical changes to unblock unattended paper trading:

1. Short-circuit `NotificationService.request_approval` when `settings.PAPER_TRADING` is true — auto-approve immediately and fire-and-forget a Telegram notification.
2. Guided in-session rotation of four exposed credentials (Gemini, Polygon, T212 key+secret, Telegram) with the new values written to the already-gitignored `.env`.
3. Add a single informative startup log line naming the active mode, pair universe, and next-market-open time.

No dependency changes, no new services, no schema changes. Total edits: one Python file, one `.env` rewrite, one new runbook doc.

## Phase 0 — Research (findings)

- **`notification_service.request_approval` current behavior** (verified at `src/services/notification_service.py:246-286`): always sends an inline-keyboard Telegram message, creates a per-call correlation-ID future, and awaits it with `asyncio.wait_for(..., timeout=300)`. On timeout returns `False`. Called by `src/monitor.py:203` before every paper execution.
- **`shadow_service.execute_simulated_trade`** (`src/services/shadow_service.py:14`): already correct for paper fills; not touched.
- **`.env` git status** (`git ls-files | grep '^.env'` → only `.env.template` tracked): secrets are NOT in git history. Rotation is driven by the fact that the values were exposed in-session, not by a git leak. No `git filter-repo` needed.
- **Monitor boot sequence** (`src/monitor.py:368-425`): runs `persistence_service.init_db()` + `initialize_pairs()` + health checks + a "Bot is active" log before entering the scan loop. The new pre-flight line fits cleanly between `setup_logging()` return and `init_db`.

## Phase 1 — Design artifacts

### `quickstart.md` (to be written in Phase 1)

Boot sequence after the fixes ship:

1. `docker compose -f docker-compose.backend.yml up -d` — start Redis + Postgres + sec-worker + execution-engine.
2. Confirm `.env` has `PAPER_TRADING=true`, `LIVE_CAPITAL_DANGER=false`.
3. `python src/monitor.py` — expect a startup line like `MODE: PAPER | DEV_MODE=false | Pair universe: 21 equity pairs | Next NYSE open: 2026-04-20 09:30 America/New_York`.
4. Watch for `SCAN [TICKER_A/TICKER_B] z=…` heartbeat within 30 seconds.
5. When a signal fires: expect Telegram notification "📝 Paper trade executed: …" (no Approve/Reject buttons) and a `SHADOW TRADE EXECUTED …` log within 2 seconds.

### `rotation-runbook.md` (to be written in Phase 1)

Sequenced walkthrough: (1) revoke Gemini key in Google AI Studio → (2) revoke Polygon key at polygon.io/dashboard/keys → (3) regenerate T212 key+secret under Trading 212 → Settings → API → (4) revoke Telegram bot via `/revoke` to BotFather and reissue, capture new token. After each step, paste the new value; the runbook updates `.env` atomically.

### Contracts

No new network contracts. The single modified function signature — `NotificationService.request_approval(trade_summary: str) -> bool` — is preserved.
