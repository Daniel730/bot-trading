# Data Model: Low-Budget Investor Suite

## Entities

### PortfolioStrategy
Represents a collection of target assets and their relative weights, associated with a risk profile.
- **ticker** (TEXT, PK): The asset symbol (mapped to T212 ID).
- **strategy_id** (TEXT): Unique ID for the strategy (e.g., 'safe', 'growth').
- **target_weight** (REAL): Percentage allocation (0.0 to 1.0).
- **risk_profile** (TEXT): 'Conservative', 'Balanced', 'Growth'.

### DCASchedule
Stores recurring investment instructions.
- **id** (TEXT, PK): Unique schedule ID.
- **amount** (REAL): Fiat amount to invest per cycle.
- **frequency** (TEXT): 'daily', 'weekly', 'monthly'.
- **day_of_week** (INTEGER): 0-6 (for weekly).
- **day_of_month** (INTEGER): 1-31 (for monthly).
- **strategy_id** (TEXT): FK to PortfolioStrategy.
- **next_run** (TEXT): ISO timestamp for the next execution.
- **is_active** (BOOLEAN): Toggle for the schedule.

### InvestmentThesis (Extended Audit Log)
The existing `audit_logs` table will be used, with the `ai_rationale` field storing the generated natural language "Investment Thesis".
- **signal_id** (TEXT): FK to signals.
- **ai_rationale** (TEXT): The "Investment Thesis" generated by the Portfolio Manager.

### FeeConfiguration
- **key** (TEXT, PK): 'max_friction_pct', 'min_trade_value'.
- **value** (REAL): The configuration value.

## Relationships
- **DCASchedule** -> **PortfolioStrategy**: A schedule dictates investments into a specific strategy.
- **PortfolioStrategy** -> **VirtualPieAsset**: Strategies manage groups of assets in the virtual pie.
- **InvestmentThesis** -> **Signal**: Every thesis is linked to a specific trade signal.

## Validation Rules
- `target_weight` sum per `strategy_id` MUST be <= 1.0.
- `max_friction_pct` default is 0.015 (1.5%).
- `amount` in `DCASchedule` MUST be >= `min_trade_value` (default $1.00).
